# Governance

NeverHuman is a community-first open source project. This document describes how decisions are made.

## Maintainers

Maintainers have write access to the repository and are responsible for reviewing PRs, triaging issues, and making release decisions.

Current maintainers are listed in [.github/CODEOWNERS](.github/CODEOWNERS).

## Decision making

**Routine changes** (bug fixes, docs, tests, dependency updates):
- Any maintainer can approve and merge

**Significant changes** (new features, architecture changes, breaking changes):
- Requires discussion in a GitHub Issue or PR
- At least one maintainer approval required
- 48-hour review window for community feedback

**Major decisions** (governance changes, licensing, telemetry/analytics, monetization):
- Require public RFC (Request for Comments) posted as a GitHub Discussion
- 14-day open comment period
- Unanimous maintainer agreement required

## Becoming a maintainer

Contributors who have made consistent, high-quality contributions over at least 3 months may be nominated for maintainership by any existing maintainer. Nomination requires consensus from all current maintainers.

## Conflict resolution

Disagreements are resolved through open discussion in GitHub issues or PRs. If consensus cannot be reached, the original project author has final say.

## Code of Conduct enforcement

See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Maintainers are empowered to enforce the code of conduct, including temporary or permanent bans from the project.
